# testing.md

このプロジェクトのテスト方針。

---

## 目的

- タイマー精度を守る
- UI崩れを防ぐ
- 設定反映を保証する
- OS連携の主要挙動を確認する

---

## テスト対象

### Rust
- タイマー状態遷移
- セッション進行
- 長休憩判定
- 設定反映
- 永続化処理

### React
- 主要画面描画
- 状態表示
- ボタン操作
- 入力反映
- 設定画面の表示分岐

### 結合
- Rust 状態の UI 反映
- 設定変更後のタイマー動作
- 通知・トレイ関連の主要フロー

---

## 最低限確認すること

- Focus 開始できる
- Pause / Resume できる
- Break に切り替わる
- セッション進行が正しい
- 長休憩条件が反映される
- 設定保存が反映される
- トレイ格納が動く
- 通知が出る

---

## 重点観点

- 見た目よりロジックの信頼性
- タイマー系境界条件
- 設定ON/OFFの分岐
- UIと状態の不一致がないこと
