# testing.md

## テスト方針

- コアの信頼性が必要な箇所は Rust 側で厚く確認する
- React 側は表示条件、操作反映、壊れやすい分岐を重点確認する
- UI とバックエンドの境界は結合観点で確認する
- 主要フローは手動確認を前提にする

## Rust の確認対象

- timer state transitions
- Focus / Short Break / Long Break の切り替え
- pause / resume / reset / skip の整合性
- セッション進行カウント
- 設定 load / save
- スリープ復帰や時刻ずれへの補正

## React の確認対象

- 状態表示がバックエンド値に追従すること
- Focus / Break の配色差が崩れないこと
- ボタン活性状態が正しいこと
- 設定 UI が入力条件を満たすこと
- 長いテキストや OS ごとのフォント差で崩れないこと

## 結合確認対象

- UI からの start / pause / resume / reset / skip が Rust に正しく伝わること
- 設定変更が保存され、再起動後も復元されること
- 通知、トレイ、ウィンドウ挙動がイベントと整合すること

## タイマー精度

- 分単位ではなく秒単位のズレも意識して確認する
- 一時停止、再開、スキップ、復帰後でも残り時間が破綻しないこと
- 長時間放置後もセッション進行が誤らないこと

## 重点シナリオ

- 初回起動から Focus 開始
- pause -> resume
- reset
- skip
- Focus 完了から Break への遷移
- 複数 Focus 後の Long Break 発火
- 設定変更後の再起動
- 通知クリックやトレイ操作からの復帰
