# tauri-react.md

## UI 実装フロー

1. `.design/design.pen` を確認し、画面の構造、余白、状態差を把握する
2. 画面を表示コンポーネント単位に分解する
3. React 側で必要な props と表示 state だけを定義する
4. Rust / Tauri 側から受け取るデータ契約を明確にする
5. スタイル実装後に Focus / Break、hover / active / disabled を確認する
6. `.design/design.pen` と見比べて差分を潰す

## Rust タイマー実装フロー

1. 状態機械を先に整理する
2. `start / pause / resume / reset / skip` をイベントとして定義する
3. 経過時刻ベースの更新戦略を決める
4. UI 通知用のイベント発行ポイントを限定する
5. 設定値とセッション進行の関係を整理する
6. テストで状態遷移と時間補正を確認する

## 設定実装フロー

1. 保存対象を最小限に定義する
2. デフォルト値とバージョン互換方針を決める
3. Rust 側で load / save の境界を作る
4. React 側は設定画面の入力と表示に集中する
5. 保存後の反映タイミングを明確にする

## OS 機能実装フロー

1. 対象機能の OS 差分を洗い出す
2. Tauri / Rust 側で差分吸収層を用意する
3. React 側には単純な command / event インターフェースだけを渡す
4. Windows / macOS で通知、トレイ、ウィンドウ挙動を個別確認する

## パフォーマンス確認観点

- idle 時 CPU が無駄に上がっていないか
- 高頻度 state 更新や interval の乱立がないか
- 起動時に不要な同期処理をしていないか
- 常駐時メモリが増え続けないか
- 重い依存や過剰な abstraction が入っていないか

## PR 前確認観点

- `.design/design.pen` との差分説明ができるか
- タイマー主制御が Rust 側に残っているか
- 型、lint、clippy、テストが通るか
- 主要フローが手動確認済みか
- 無関係な変更が混ざっていないか
