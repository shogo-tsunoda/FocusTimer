# coding-standards.md

このファイルは、このリポジトリの実装規約を定義します。

---

## 共通方針

- 小さく変更する
- 変更理由を明確にする
- 可読性を優先する
- ただし冗長にしない
- 未使用コードを残さない
- 使わない依存を追加しない

---

## TypeScript / React

### 必須
- strict mode
- props / state の型を明示
- 小さなコンポーネントに分割
- 表示とロジックの責務を分離

### 推奨
- ローカル state 優先
- derived state を増やさない
- 必要箇所だけ memo / useMemo / useCallback を使う

### 禁止
- any の濫用
- context の濫用
- 無意味な custom hook の乱立
- 大規模 UI ライブラリの安易な導入

---

## Rust / Tauri

### 必須
- clippy clean
- 明確な状態遷移
- エラーを握りつぶさない
- 不要 clone を避ける

### 推奨
- シンプルなデータ構造
- OS差異を分離
- 小さなモジュール単位に整理

### 禁止
- 必要性の薄い async の乱用
- 不要な共有状態
- 過剰なイベント駆動
- 便利そうという理由だけの plugin 追加

---

## UI 実装

- Pencil を正本とする
- タイマー表示を主役にする
- Focus は青、Break は緑
- 過剰な animation を入れない
- 余白と視認性を崩さない

---

## 依存追加ルール

依存を追加する場合は、必ず以下を明示する。

1. 何のために必要か
2. 代替手段がないか
3. サイズ・性能・保守性への影響

重いライブラリは原則として避ける。

---

## コメント方針

- 「なぜ」を補足するコメントは可
- 「何をしているか」だけのコメントは避ける
- 自明なコードにコメントを足しすぎない

---

## リファクタ方針

- 修正対象に近い範囲に留める
- 大きな整理は別タスクとして分離する
- 機能変更と大規模リファクタを混ぜない
