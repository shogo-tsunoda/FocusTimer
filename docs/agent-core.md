# agent-core.md

このファイルは、すべてのエージェントに共通する中核ルールです。

---

## プロジェクトの本質

このアプリは、単なるタイマーではなく、
**軽量・高速・シンプルなデスクトップ用ポモドーロユーティリティ**です。

重視する価値:

- すぐ使える
- 迷わない
- 見た目が美しい
- 常駐に向く
- 無駄がない

---

## 最重要の優先順位

判断に迷った場合は、以下の順で優先してください。

1. Pencil デザインへの忠実さ
2. CPU / メモリ効率
3. 起動速度
4. タイマー精度
5. Windows / macOS 整合性
6. 可読性・保守性
7. 新機能追加

---

## Pencil を唯一の正本とする

- `.pen` ファイルとエクスポート画像を UI 実装の正本とする
- 勝手に UI を再設計しない
- 見た目を変更する場合は、理由を明示して提案する

許可される微修正:

- OS差異の吸収
- 実装制約上の最小限の調整
- アクセシビリティ改善
- レイアウト崩れ回避の微調整

---

## アプリの性格

このアプリは以下の性格を持つ。

- デスクトップ特化
- ローカル完結
- 単機能
- ミニマル
- 高速起動
- 常駐向き

「便利そうだから」で機能や依存を増やしてはいけない。

---

## 実装の基本思想

### フロント
- 表示中心
- 状態は必要最小限
- 無駄な再レンダリングを避ける

### Rust / Tauri
- タイマー主制御
- OS連携
- 通知
- トレイ
- 永続化

### UI
- 残り時間が主役
- Focus は青
- Break は緑
- 視認性優先
- 派手な演出は禁止

---

## 変更の出し方

大きな変更を行う場合は、簡潔に以下を示す。

- 変更内容
- 変更理由
- パフォーマンス影響
- UI再現性への影響
- 依存追加の有無

---

## 禁止事項

- Pencil からの独断逸脱
- 過剰設計
- 重い依存の安易な導入
- 毎フレーム更新前提の実装
- 修正と無関係な大規模リファクタ
- コメント過多で冗長なコード
- 不要な abstraction の乱用

---

## 品質の最低ライン

### TypeScript
- strict mode 前提
- any を安易に使わない
- 未使用コードを残さない

### Rust
- clippy clean
- 不要 clone を避ける
- 状態遷移を明快に保つ

### 共通
- 小さく変更する
- 読みやすく保つ
- 依存は厳選する
