# agent-core.md

## 目的

このプロジェクトは、`.design/design.pen` を正本として実装する、軽量で高速なポモドーロデスクトップアプリを作るためのものです。  
AI エージェントは、この文書を共通原則として扱い、判断が割れる場面では本ファイルを優先してください。

## プロジェクトの本質

- 主役はタイマー表示とセッション進行であり、多機能化は目的ではない
- 常駐ユーティリティとして邪魔にならない軽さを最優先する
- UI は派手さより視認性、余白、情報階層、状態差を重視する
- Windows / macOS の両方で自然に使えることを前提にする

## 優先順位

1. `.design/design.pen` との整合
2. タイマーの正確性と安定性
3. 低 CPU / 低メモリ / 高速起動
4. 責務分離の明確さ
5. シンプルで理解しやすい UX
6. 型安全性と保守性

## デザインの正本

- UI 実装の正本は `.design/design.pen`
- レイアウト、余白、情報階層、色、主従関係を忠実に再現する
- Focus は青系、Break は緑系という状態差を守る
- 過剰なアニメーションは追加しない
- 視認性を損なう装飾や独自解釈を入れない

### 許容される調整

- Windows / macOS の標準 UI 慣習に合わせるための最小限の差分
- 実装上必要な hover / active / disabled などの状態定義
- アクセシビリティやテキスト崩れ回避のための微調整

### 禁止

- `.design/design.pen` を見ずに UI を再構成すること
- 色、余白、情報の強弱を勝手に変更すること
- パフォーマンスに不利な演出を追加すること

## エージェント共通原則

- 変更前に「何を変えるか、なぜ変えるか、性能影響、依存追加有無」を簡潔に整理する
- 小さい変更単位で進める
- 無関係な修正を混ぜない
- 実装後は型、lint、テスト、主要フロー確認を行う
- 依存追加は最後の手段とし、理由を明文化する

## アーキテクチャ原則

- タイマー主制御は Rust / Tauri 側が持つ
- React は表示層中心で、状態は最小限にとどめる
- OS 連携は Rust / Tauri に集約する
- UI の都合だけでバックエンド責務を増やさない
- 毎フレーム更新前提の設計は禁止する

## 禁止事項

- フロントエンドだけでタイマーを主制御すること
- 重い UI フレームワークや不要なランタイム依存の導入
- 大規模な抽象化や先回りした汎用化
- 型安全性を崩す回避策の常用
- 一回の作業に複数の目的を混ぜること

## 品質の最低ライン

### TypeScript / React

- strict mode 前提
- `any` や無理な型アサーションを常用しない
- 不要な再レンダリングを避ける
- コンポーネントは責務が読める単位に保つ

### Rust / Tauri

- `cargo clippy` で重大な警告を残さない
- 状態遷移は明示的に表現する
- ブロッキング処理や重い処理を UI イベントに直接ぶら下げない

### 全体

- 主要フローが壊れていないこと
- 設定保存と復元が一貫していること
- 通知、トレイ、ウィンドウ挙動が OS ごとに破綻しないこと
