# Design Implementation Skill

## 目的

`.design/design.pen` を UI 実装の正本として扱い、レイアウト、余白、色、情報階層を忠実に React 側へ落とし込むためのスキルです。

## 前提

- UI の正本は `.design/design.pen`
- デザインを勝手に再設計しない
- タイマー表示を最優先で目立たせる
- Focus は青系、Break は緑系を守る

## 見るべき観点

### 余白

- 外側余白と内側余白を分けて観察する
- セクション間の距離と、同一グループ内の距離を混同しない
- 中央揃えと視線誘導の基準線を崩さない

### 階層

- 何が主役かを最初に決める。このアプリではタイマー表示が主役
- 補助情報や設定導線は主役を邪魔しない位置に置く
- ボタンの主従、視線順、コントラスト差を再現する

### 色

- Focus は青系
- Break は緑系
- 状態差は色だけでなく、ラベルやコントラストでも分かるようにする
- 彩度や明度を勝手に変えて UI 印象を変えない

### 状態差

- hover / active / disabled を実装する
- 実行中、停止中、休憩中の違いを明確に出す
- 非活性時に情報が消えすぎないようにする

## 実装原則

- まずレイアウト骨格を作り、その後に装飾を足す
- 数値は感覚で置かず、`.design/design.pen` から一貫性のある値を抽出する
- UI 部品は再利用できても、正本を崩す抽象化はしない
- アニメーションは必要最小限に留める

## 禁止事項

- デザインを見ずに一般的なポモドーロ UI に寄せること
- 余白、色、主従関係を独断で変更すること
- 実装の楽さだけで部品構造を変えること
- 見た目を合わせるために意味のない DOM や state を増やすこと

## 提案が必要な場合

`.design/design.pen` に対して実装上の懸念がある場合は、以下をセットで提案すること。

- 問題点
- 現状デザインへの影響
- 代替案
- パフォーマンス影響
- Windows / macOS への影響
