# Tauri Performance Skill

## 目的

Tauri アプリを常駐向きの軽量アプリとして維持するための性能指針です。

## 最重要原則

- 低 CPU
- 低メモリ
- 高速起動
- 不要依存を避ける
- フロントエンドに重い処理を持たせない

## plugin の扱い

- plugin は必要性が明確なものだけ使う
- 代替手段が Tauri 標準や軽い自前実装で足りるなら追加しない
- 追加時は起動時間、バイナリサイズ、保守性への影響を整理する

## 起動速度

- 起動時に不要な設定読み込みや重い初期化をしない
- 初回表示に不要な処理は遅延させる
- フロントで巨大データや重いライブラリを初期ロードしない

## idle 時 CPU

- ポーリングを最小化する
- 毎フレーム更新は禁止
- タイマー更新は必要十分な粒度で行う
- バックグラウンド時の再描画や不要イベントを抑える

## 常駐時メモリ

- 常時保持する状態を絞る
- 大きなキャッシュや履歴保持を避ける
- イベント購読解除漏れを防ぐ

## フロントの原則

- 高頻度処理は Rust 側で完結できるかを先に検討する
- 表示だけでよい情報を必要以上に保持しない
- ランタイム依存をむやみに増やさない

## チェック観点

- 起動後放置時の CPU
- セッション実行中の CPU 変動
- メモリ増加傾向
- 不要な interval / timeout / event listener
- 重い依存追加の有無
