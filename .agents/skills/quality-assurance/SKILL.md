# Quality Assurance Skill

## 目的

UI 忠実性、パフォーマンス、責務分離、型安全、依存管理の観点で変更をレビューするためのスキルです。

## レビューの優先順位

1. `.design/design.pen` との乖離
2. タイマー主制御が Rust 側に残っているか
3. パフォーマンス悪化がないか
4. 型安全性と状態遷移の明確さ
5. 不要依存や無関係な変更の混入

## UI 忠実性観点

- タイマーが主役になっているか
- Focus / Break の色差が守られているか
- 余白、整列、階層が崩れていないか
- hover / active / disabled が抜けていないか

## パフォーマンス観点

- 毎フレーム更新がないか
- React 側の高頻度再レンダリングがないか
- 起動時に重い処理が増えていないか
- plugin や依存の追加が妥当か

## 責務分離観点

- React にタイマー制御が漏れていないか
- Rust に UI 都合ロジックが混ざっていないか
- OS 差分処理が適切な層にあるか

## 型安全 / 保守性観点

- TypeScript strict を崩す実装がないか
- Rust の状態遷移が読めるか
- 無関係なリファクタが混ざっていないか
- コメントと命名で意図が追えるか
