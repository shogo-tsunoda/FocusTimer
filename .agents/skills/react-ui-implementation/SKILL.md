# React UI Implementation Skill

## 目的

React + TypeScript で、軽量で保守しやすく、`.design/design.pen` に忠実な UI を実装するためのスキルです。

## 基本方針

- React は表示層中心
- state は最小限
- タイマー主制御は持たない
- 不要な再レンダリングを避ける
- 重い UI ライブラリを避ける

## コンポーネント分割

- 画面単位、セクション単位、再利用が自然な UI 部品単位で分ける
- 過剰な抽象化より、読みやすい構造を優先する
- props は表示に必要な最小限に絞る
- derived state を増やさず、表示時に計算できるものはその場で扱う

## state 最小化

- サーバーや Rust 側が真実を持つ state を複製しない
- 一時的なフォーム入力や UI 開閉状態だけをローカル state に置く
- 複数箇所で共有する必要がない限り global state を増やさない

## 再レンダリング最適化

- 高頻度更新値を広い範囲に伝播させない
- タイマー残り時間表示は必要な範囲だけ更新する
- 無駄な object / function 再生成でツリー全体を再描画させない
- まず構造を見直し、最後に必要なら最適化 API を検討する

## スタイリング

- `.design/design.pen` の余白と階層を優先する
- Focus は青系、Break は緑系
- 見た目を整えるためだけの重い CSS フレームワークは避ける
- hover / active / disabled を最初から考慮する

## 避けること

- 巨大コンポーネント化
- 先回りした汎用 UI ライブラリ化
- 状態を React 側に寄せすぎること
- 描画都合でバックエンド契約を曖昧にすること
